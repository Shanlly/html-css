A Little History

Prolog was first conceived in 1970 by Alain Colmerauer and finally was created in 1972 by Alain and Philippe Russel in France. It wasn’t distributed until 1974-75. Prolog was based on Robert Kowalski’s procedural interpretation of Horn clauses.

The Purpose, Limitations and Type of language

The pure Prolog was originally used only for resolution theorem prover with Horn clauses of the form. Prolog is a compiling code that uses logical base. Prolog is now widely used for research and education for theorem proving, automated planning as well as used for natural language processing and AI which is was originally build.
It has the ability to run programs “backwards and forwards” which is one of it’s main attractions. Prolog is suited for specific tasks like database searching or filling in templates which use rule-based logical queries. It is mostly a declarative language but isn’t purely because of some of the constructs it uses and isn’t a strongly typed language.

Here is an Example of Prolog

% select(X,HasXs,OneLessXs) deletes a single X from HasXs
select(X, [X|Xs], Xs).
select(X, [Y|Ys], [Y|Zs]) <- select(X, Ys, Zs).
% insert(X,OneLessXs, HasXs) inserts X into OneLessXs at a non-deterministic position
insert(X, Ys, Zs) <- select(X, Zs, Ys).
% permutation1(Input, Output)
% non-deterministically select the head of the output, permute the remaining input and use as the tail
permutation1(Xs, [Z|Zs]) <- select(Z,Xs,Ys), permutation(Ys,Zs).
permutation1([], []).
% permutation2(Input,Output)
% permutate the tail of the input, then insert the head of the input into a non-deterministic position
permutation2([X|Xs], Zs) <- permutation(Xs,Ys), insert(X,Ys,Zs).
permutation2([], []).
